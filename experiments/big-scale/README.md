# big-scale

Large-scale variant of [`forever`](../forever/) — 50 satellites (OneWeb
constellation) + 7 ESTRACK ground stations.

Satellites use the `nano-sat` hardware profile (200m CPU, 256Mi RAM) so the
full constellation fits on a single cluster. Ground stations keep
`ground-station-hwdef`.

The primary engine is **EDFS** (`./edfs`); a TUS overlay (`./tus`) is also
generated for side-by-side comparison.

## Scenario

- Every satellite runs `yass-agent-periodic` and creates a **5 MB** file
  every ~5 minutes. `CHECK_INTERVAL_SECONDS` is varied per satellite
  (270..329s) so the production phases spread out within a few
  iterations rather than firing in lock-step.
- Every satellite also carries a **recurring hardware fault stream**
  (non-`Destroy`) with mean interval 5m, jitter 50%, duration 30s
  ± 50%. The fault type rotates across the constellation
  (`NetworkBandwidthReduced` → `NetworkFailure` → `DiskFull` →
  `DiskFailure`), with a per-sat seed for reproducibility.
- Ground stations run `yass-agent-receive-only`. They do not produce
  files and do not carry hardware events.
- `maxDuration: 999999h` — the experiment is open-ended; tear down with
  `kubectl delete -k …` when done.

## Layout

Satellite TLEs come from a snapshot of the OneWeb constellation
(`tools/oneweb-tle-snapshot.txt`, fetched from celestrak.org). The first
50 entries are used. The total FsNode count (sats + GS) must stay ≤ 200,
the geo-calculator binary's compile-time `MAXSAT` limit.

`base/01_layout.yaml`, `tus/{02_experiment_definition,03_experiment}.yaml`
and `edfs/{02_experiment_definition,03_experiment,kustomization}.yaml`
are auto-generated by `tools/gen.py`. To regenerate after refreshing the
TLE snapshot:

```shell
curl -s "https://celestrak.org/NORAD/elements/gp.php?GROUP=oneweb&FORMAT=tle" \
  -o tools/oneweb-tle-snapshot.txt
python3 tools/gen.py
```

## Prerequisites

Apply `nano-sat` and `ground-station-hwdef` HardwareDefinitions to the
cluster (once per environment):

```shell
kubectl apply -f ../_common_/hardware_specs.yaml
```

## Apply

```shell
kubectl apply -k ./edfs    # primary
# or
kubectl apply -k ./tus     # comparison
```

## Resource budget (rough)

50 satellite pods × (`nano-sat` 256Mi + engine containers +
world-controller) + 7 GS pods + control plane. EDFS adds two extra
engine containers per pod (`edfs-engine-node` and `edfs-engine-proxy`)
on top of the agent + world-controller, so its footprint is noticeably
larger than the TUS variant.

## Delete

```shell
kubectl delete -k ./edfs
# or
kubectl delete -k ./tus
```
