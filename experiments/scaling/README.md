# scaling — Delivery time vs constellation size

For a fixed producer satellite (`oneweb-0008`) and the full ESTRACK
ground network (7 GSes from `forever/base/01_layout.yaml`), measure how
"time to deliver to any GS" changes as `n` (number of satellites in the
swarm) grows from 1 to 8 — once with the **TUS** engine and once with
**EDFS**.

Expected: TUS is roughly flat (only the producer's own visibility of
*some* GS matters), EDFS decreases with `n` (extra satellites are libp2p
relays — they multiply the paths to a GS).

## Layout

```
producer (oneweb-0008)       periodic-agent, 10 MiB every 120 s
relays (relay-01..NN-1)      receive-only-agent (inert for TUS;
                             libp2p relays for EDFS)
7 ESTRACK GSes               receive-only-agent
  new-norcia, kiruna, redu,
  cebreros, santa-maria,
  kourou, malargue
```

The 7 GSes mirror `forever/base/01_layout.yaml`. TUS engine's
`GROUND_STATIONS` env lists all 7 (matching `forever/tus`); TUS uploads
to the first reachable GS and deletes locally. EDFS bootstrap is
`estrack-kiruna` (matching `forever/edfs`).

Relay TLEs share the producer's orbital plane (87.9° inclination, OneWeb
altitude) but are evenly spread in RAAN and mean anomaly by 360/8°. They
are generated by [`tools/gen-relay-tles.py`](tools/gen-relay-tles.py)
and committed in [`relay-tles.yaml`](relay-tles.yaml).

## Layout structure

```
scaling/
├── relay-tles.yaml              # generated relay orbit data
├── _engines/{tus,edfs}/         # Experiment + engine containers
├── n{01,02,03,05,08}/
│   ├── base/                    # per-n Layout + ExperimentDefinition
│   └── {tus,edfs}/              # namespace + kustomization overlay
└── tools/
    ├── gen-relay-tles.py        # one-off TLE generator
    ├── gen-variants.py          # regenerates per-n YAMLs from relays
    └── run-scaling.sh           # sequential runner for the full sweep
```

The per-n files are produced by `gen-variants.py`. Edit the generator,
not the generated files (they are overwritten on the next run).

## Run

Single variant:

```sh
kubectl apply -k yass-experiments/experiments/scaling/n03/edfs
# … soak …
kubectl delete -k yass-experiments/experiments/scaling/n03/edfs
```

Full sweep (10 runs, sequential — ~70 h with default 6 h soak):

```sh
yass-experiments/experiments/scaling/tools/run-scaling.sh
```

The runner writes `tools/run-ids.tsv` with one row per run:

```
n  engine  namespace        run_id                                 started_at
1  tus     scaling-n01-tus  scaling-experiment_20260516T143012Z    2026-05-16T14:30:12Z
1  edfs    scaling-n01-edfs scaling-experiment_20260516T220045Z    2026-05-16T22:00:45Z
…
```

Override the sweep with env vars, e.g. dry-run on a single short variant:

```sh
N_VALUES="2" ENGINES="edfs" SOAK_SECONDS=600 \
  yass-experiments/experiments/scaling/tools/run-scaling.sh
```

## Compare

Per `n`, pair the two `run_id`s from `run-ids.tsv` and feed them to
`yass-compare`:

```sh
yass-compare \
  --prometheus http://prometheus.yass-system.svc:9090 \
  --experiment scaling-experiment \
  --tus-run  scaling-experiment_20260516T143012Z \
  --edfs-run scaling-experiment_20260516T220045Z \
  --window 7h \
  --out report-n03.md
```

The headline panel is `yass_file_delivery_seconds` filtered to
`target_fsNode=~"estrack-.*"` (delivery to any ESTRACK GS).

## Regenerating variants

If you change the relay count, file size, target GS, producer, etc.:

```sh
cd yass-experiments/experiments/scaling
python3 tools/gen-relay-tles.py   # only if relay count > 7
python3 tools/gen-variants.py
```

## Caveats

- `maxDuration: 168h` (7 days) is set on the ExperimentDefinition so a
  manual run won't auto-stop early. `run-scaling.sh` enforces its own
  timing via `SOAK_SECONDS`.
- The EDFS bootstrap peer is `estrack-kiruna`, matching `forever/edfs`.
- `n=8` plus 7 GSes = 15 fsNode pods on the KinD node. Use the CPU-fake
  trick from [`NOTES.md`](../../../NOTES.md) (24-CPU patch) before
  running the larger cases on a 12-CPU host.
- The simulator's geo-clock runs in real time. A "6-hour soak" really
  takes 6 wall-clock hours and covers ~4 LEO orbits.
